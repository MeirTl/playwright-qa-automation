# Тест-план: Автоматизация Task Manager App

## 1. Введение
Данный документ описывает стратегию автоматизированного тестирования для приложения Task Manager. Цель — обеспечить регрессионное тестирование критического функционала.

## 2. Область тестирования (Scope)
### Включено:
- Модуль авторизации (позитивные/негативные сценарии).
- Управление задачами (создание, удаление, отображение списка).
- Поиск и фильтрация задач.
- Валидация экранных форм и сообщений об ошибках.

### Исключено:
- Тестирование производительности (Load/Stress).
- Реальное взаимодействие с БД (используется мокирование данных).

## 3. Технологический стек
- **Инструмент**: Playwright (JS).
- **Браузеры**: Chromium, Firefox, Webkit.
- **CI/CD**: GitHub Actions.

## 4. Стратегия тестирования
- **Уровни**: UI функциональные тесты + API Interception для изоляции.
- **Подход**: Page Object Model (POM).
- **Данные**: Статические тестовые данные в коде + динамические генерации (Date.now()).

## 5. Критерии приемки (Acceptance Criteria)
- Все тесты проходят успешно (Green Pipeline).
- Для каждого падения сгенерирован скриншот и видео.
- Код соответствует стандартам ESLint.

## 6. Риски
- **Flaky tests**: Минимизируются за счет использования `locator.waitFor()` и авто-ожиданий.
- **Environment**: Локальный сервер `http-server` должен быть запущен перед тестами (автоматизировано в CI).
